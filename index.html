Bulk export notes

Bulk randomises the generator engine, Primary Phase, Shape 1, Shape 2, wavetable type, FM/PWM/AM, FM pitch range, source/result windows, and START/END patches. Filters are reset to Off so random batches are less likely to disappear into silence.

Bulk uses the render settings shown on this tab: length, sample rate, bit depth, morph render mode, FFT anchor count, DC removal, normalisation, and headroom. Those settings are shared with the Morph tab. Direction and download mode are Bulk-only controls.

Locked in bulk mode: there is no per-iteration audition, no manual START/END editing during a batch, no separate START/END FM-range toggle, and no per-file morph-mode variation unless you change the global Morph Render Mode before starting the batch.

Notes

Fractal Wavetable is a browser-native, single-file wavetable generator. It is designed for making loopable oscillator material, long evolving WAV files, drones, noisy transitions, and sound-design source material.

Usage Notes

Pressing ESC stops audio and cancels active rendering. Leaving a tab also mutes the current audition.

Pitch is shared across Playground, Morph, and Bulk. The frequency slider is logarithmic; note/octave controls and MIDI note input set the same frequency. The selected-note readout shows the exact equal-tempered note frequency, even when the typed frequency is between notes. MIDI input is gated: note-on starts or retunes the current drone, note-off releases it, and the most recently held note has priority. MIDI input requires browser Web MIDI support and usually works from local files in Chromium-based browsers.

The generator engines are clean-room DSP experiments built from ordinary waveforms, phase warping, folding, smoothing, saturation, bitwise/quantised collision, and spectral table reconstruction. Wavetable Type is the collision/combine stage.

Primary Phase sets the main phase/seed position. Shape 1 and Shape 2 shape the two internal source waves. Smooth engines are more rounded and root-stable; Crunchy engines push harder phase warp, folding, and digital grit.

Visualisation note: FM/PWM/AM views and the spectrum analyser show a post-filter preview. Base table shows the generated table before oscillator modulation and filtering. The spectrum analyser is a cyclic single-cycle analyser, not a room/microphone analyser.

Morph render mode changes the transition character. Full Interpolation favours digital noise. Crossfade favours analogue warmth but can phase or beat. Wavetable modes blend oscillator tables before playback. FFT modes rebuild tables from harmonic data; more anchors use more CPU and add more internal motion.

Export notes: WAV files are generated in browser memory before download. Very long renders can stress RAM and browser Blob/download limits. ZIP exports drop their temporary references after the download is triggered, but the browser decides when memory is actually reclaimed.
